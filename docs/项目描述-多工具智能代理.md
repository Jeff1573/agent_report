### 基于 ReAct/LangGraph + MCP 的多工具智能代理 | 2025.03 - 至今

- 项目简介：基于 ReAct 思维—行动闭环与 MCP 工具生态，在 Electron 上交付跨平台桌面智能体，覆盖检索—推理—工具调用—可视化闭环，兼容 OpenAI Chat Completions 协议（结构化输出/工具调用/流式）。

- 技术栈：LangChain.js v0.3 + LangGraph、React 18、TypeScript、Electron、Chromadb、MCP、Ant Design

- 主要功能
  - 跨平台桌面：Electron 打包 Windows/macOS/Linux；主/渲染进程解耦，`contextIsolation: true`、禁用 `nodeIntegration`、IPC 白名单。
  - 可视化与配置：模型/密钥/代理与 `baseURL` 自定义；JSON 导入/导出；会话历史持久化；流式能力自检（首 token 时延/支持度）与结果入库。
  - 协议兼容：结构化输出/JSON 模式、函数/工具调用、逐 token 流式；支持自定义认证头与兼容/中转端点。
  - ReAct 编排：LangGraph ReAct + AgentExecutor 双模；自动工具选择、思考/调用/结果/答案四段式链路可视化；Memory 与 Postgres Checkpointer 可切换。
  - RAG：Chromadb 检索（similarity/MMR），默认客户端 MMR 重排与失败回退；答案端自动补全“参考来源”。
  - MCP 工具：多服务器加载与热插拔；调用次数上限、超时与重试的韧性包装；`mcp.json` 内容变更热重载 Agent Runtime。

- 核心职责与成果
  - 设计可插拔 ReAct 运行时：在 `agent/runtime/index.ts` 实现 LangGraph 与 AgentExecutor 双模式，按场景选择；`tool_choice: auto` 降低误触发。
  - 工具编排与容错：对 MCP 工具注入“总次数上限（默认100）+ 45s 超时 + ≤5 次重试”包装，失败隔离、不中断主流程。
  - 流式能力验证：在 `agent/llm/streaming-validator.ts` 与桌面端 `settingsService.validateModelStreaming` 实现“支持度/首包时延/耗时”检测并入库。
  - 可观测与可视化：渲染端 `ExecutionStepsPanel` 将思考/调用/结果/答案流程化展示，提升问题复现与调优效率。
  - RAG 工具化：`kb_search` 统一 JSON 输出与引用索引，支持 where 过滤、客户端 MMR、失败退化重试，自动补全参考来源段落。
  - Electron 工程化与安全：主/渲染进程解耦，`preload` 白名单 API；`mcp.json` 变更哈希监控 + 防抖热重载（广播渲染端提示）。

- 可量化成果（当前实现可落地采集/填写）
  - 流式体验：首 token 延迟（p50/p95）= [X ms]/[Y ms]；流式可用率（已配置模型）= [M/N]。
  - 工具稳定性：工具调用成功率 [A% → B%]；超时/失败导致中断率 [-C%]；MCP 配置热重载时延 ≤ ~2s。
  - 检索与回答质量：MMR(client) 使用占比 [P%]；退化重试触发率 [R%]；答案含引用覆盖率 [S%]。
  - 说明：上述指标分别来源于“流式验证结果入库”“MCP 调用包装日志”“`kb_search` 输出与答案引用检测”，可脚本聚合或页面展示。

- 难点与解决
  - 兼容非标准 OpenAI 端点：统一 `baseURL` 尾斜杠与自定义认证头策略，规避 JSON 模式/流式差异。
  - 工具失效与抖动：超时/重试/总量限制组合，配合“失败不致命”降级与事件级可观测，保障会话完成率。
  - 上下文长度与引用：RAG 上下文预算裁剪与引用索引映射，保证答案可追溯性。
