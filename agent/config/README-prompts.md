# 系统提示词配置指南

## 概述

系统提示词是指导LLM进行多步推理和工具调用的关键配置。通过精心设计的系统提示词，Agent能够：

- 🔄 理解复杂任务需要多步操作
- 🛠️ 主动调用多个工具获取信息
- 🧠 基于前一步结果规划下一步行动
- 📊 整合多个信息源形成完整答案

## 当前系统提示词特性

### 多步思考流程
```typescript
const SYSTEM_PROMPT = `你是MindForge智能助手...

## 工作原则

### 1. 多步思考流程
当遇到复杂问题时，你应该：
1. **分析问题**：理解用户需求，识别关键信息需求
2. **制定计划**：将任务分解为可执行的步骤
3. **执行步骤**：按计划调用相应工具获取信息
4. **整合结果**：基于工具返回的信息形成完整答案
5. **验证完整性**：检查是否还有遗漏或需要补充的信息
`;
```

### 工具使用策略
- **优先使用MCP工具**：对于实时信息和专业内容
- **知识库检索优先**：当问题涉及内部代码/文档信息时，优先调用 `kb_search`。若问题指向特定文件/模块，请在工具参数中传入 `where`（例如 `{ filePath: "E:\\jf\\mindForge_re\\agent\\moda.py" }`），必要时调整过滤条件以确保命中。
- **多次调用**：如果一次调用无法获取足够信息，可以多次调用不同工具
- **信息验证**：从多个来源验证重要信息的准确性
- **上下文理解**：基于已获取的信息决定是否需要进一步调用工具

## 使用场景示例

### 1. 研究类问题
**用户查询**： "分析当前AI领域的最新发展趋势"

**Agent行为**：
1. 调用 `web_search_exa` 搜索最新AI新闻
2. 调用 `get_code_context_exa` 查找相关技术文档
3. 调用 `kb_search` 查询内部知识库
4. 整合所有信息形成综合分析

### 2. 技术问题解决
**用户查询**： "如何优化我的React应用的性能？"

**Agent行为**：
1. 调用 `get_code_context_exa` 搜索React性能优化最佳实践
2. 调用 `web_search_exa` 查找最新的性能优化技术
3. 基于搜索结果提供具体建议

### 3. 多领域综合查询
**用户查询**： "比较Python和JavaScript在数据科学领域的优缺点"

**Agent行为**：
1. 分别搜索Python和JavaScript的数据科学生态系统
2. 比较性能、库支持、社区活跃度等指标
3. 提供基于数据的客观分析

## 配置自定义提示词

### 修改系统提示词
编辑 `agent/config/prompts.ts` 中的 `SYSTEM_PROMPT`：

```typescript
export const SYSTEM_PROMPT = `你的自定义系统提示词...

// 添加特定领域的指导原则
// 强调多步思考的重要性
// 指定工具使用优先级
`;
```

### 添加领域特定指令
```typescript
// 例如，为代码开发添加特定指导
const DEVELOPER_SYSTEM_PROMPT = `
${SYSTEM_PROMPT}

## 代码开发专项指导
- 优先使用 get_code_context_exa 查找官方文档
- 对于复杂问题，建议分解为多个子任务
- 总是验证代码示例的可运行性
- 提供完整的错误处理建议
`;
```

## 调试和优化

### 观察Agent行为
通过 `streamEvents` 模式观察Agent的思考过程：

```typescript
const events = runtime.streamEvents(query, { summary: true });
for await (const event of events) {
  if (event.type === 'tool-call') {
    console.log('Agent调用工具:', event.name);
  }
  if (event.type === 'assistant-message') {
    console.log('Agent思考:', event.content);
  }
}
```

### 优化提示词
1. **具体性**：明确说明期望的思考步骤
2. **工具指导**：清晰说明何时使用哪种工具
3. **输出格式**：指定期望的回答结构
4. **边界情况**：处理可能出现的异常情况

## 最佳实践

1. **保持简洁**：避免过长的提示词影响推理效率
2. **结构化**：使用清晰的标题和编号
3. **具体化**：提供具体的使用场景和预期行为
4. **迭代优化**：根据实际使用效果调整提示词
5. **测试验证**：在不同场景下验证提示词效果

## 故障排除

### 常见问题
1. **Agent不调用工具**：检查提示词中是否明确了工具使用场景
2. **过度调用**：调整提示词以避免不必要的工具调用
3. **信息整合差**：加强结果整合和验证的指导

### 调试技巧
- 使用详细日志观察Agent决策过程
- 测试单个工具调用确认工具正常工作
- 逐步调整提示词并观察效果变化
